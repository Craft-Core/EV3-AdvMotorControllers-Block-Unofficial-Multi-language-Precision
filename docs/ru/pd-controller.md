# OFDL PD Controller — Руководство по использованию

Контроллер слежения за линией PD (Пропорционально-Дифференциальный), который считывает показания двух датчиков цвета и управляет двумя моторами для удержания робота по центру линии.

---

## Концепция

```
error      = P1 − P2
derivative = error − prev_error

correction = Kp × error + Kd × derivative

left_motor  = Power + correction
right_motor = Power − correction
```

- **Только P-управление** (режимы `P_*`): использует только `Kp × error`
- **PD-управление** (режимы `PD_*`): использует как `Kp`, так и `Kd`
- **PDpwr** (режимы `PDpwr_*`): то же, что PD, но читает Power из глобальной конфигурации

---

## Настройка

### Шаг 1 — Блок конфигурации (запускается один раз перед циклом)

| Параметр | Описание | Типичное значение |
|----------|----------|-------------------|
| **Ports** | Порты моторов (B+C) | `1.B+C` |
| **Kp** | Пропорциональный коэффициент | `0.3` |
| **Kd** | Дифференциальный коэффициент | `0.7` |
| **Power** | Базовая скорость мотора (−100 до 100) | `50` |

### Шаг 2 — Блок управления (запускается на каждой итерации цикла)

| Параметр | Описание |
|----------|----------|
| **P1** | Сырое значение левого датчика цвета |
| **P2** | Сырое значение правого датчика цвета |

---

## Режимы

| Режим | Категория | Описание |
|-------|-----------|----------|
| `Configuration` | — | Задать Kp, Kd, Power, Ports (вызвать один раз) |
| `P_Large` | P\_byVars | P-управление, порты крупных моторов |
| `P_Medium` | P\_byVars | P-управление, порты средних моторов |
| `P_twoRev` | P\_byVars | P-управление, обратное направление моторов |
| `PD_Large` | PD\_byVars | PD-управление, порты крупных моторов |
| `PD_Medium` | PD\_byVars | PD-управление, порты средних моторов |
| `PD_twoRev` | PD\_byVars | PD-управление, обратное направление моторов |
| `PDpwr_Large` | PDpwr\_byVars | PD + настраиваемая мощность, крупные моторы |
| `PDpwr_Medium` | PDpwr\_byVars | PD + настраиваемая мощность, средние моторы |
| `PDpwr_twoRev` | PDpwr\_byVars | PD + настраиваемая мощность, обратное направление |

---

## Типичная структура цикла

```
[Configuration: Ports=B+C, Kp=0.3, Kd=0.7, Power=50]

Loop:
  [Read Color Sensor 1] → P1
  [Read Color Sensor 2] → P2
  [PD_Large: P1, P2]
```

---

## Советы

- Начните с `Kp=0.3, Kd=0.7`. Увеличьте `Kp` для более быстрой коррекции; увеличьте `Kd` для уменьшения колебаний.
- Используйте режимы `PDpwr_*`, когда нужно динамически изменять скорость (например, в сочетании с контроллером ускорения).
- Используйте `P_twoRev` / `PD_twoRev`, если левый и правый моторы вашего робота установлены в противоположных ориентациях.
