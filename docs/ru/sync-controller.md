# OFDL MoveSync Controller — Руководство по использованию

Поддерживает одинаковую скорость двух моторов при движении по прямой, непрерывно корректируя дрейф энкодеров с помощью P-контроллера.

---

## Концепция

```
drift      = E1 − E2
correction = Kp × drift

left_motor  = Power1 − correction
right_motor = Power2 + correction
```

Без синхронизации даже одинаковые моторы при одинаковой заданной скорости со временем начнут расходиться из-за механических отличий. Этот блок устраняет такой дрейф.

---

## Настройка

### Шаг 1 — Блок конфигурации (запускается один раз перед циклом)

| Параметр | Описание | Типичное значение |
|----------|----------|-------------------|
| **Ports** | Порты моторов | `1.B+C` |
| **Kp** | Коэффициент коррекции синхронизации | `0.12` |
| **Power1** | Базовая скорость левого мотора (−100 до 100) | `50` |
| **Power2** | Базовая скорость правого мотора (−100 до 100) | `50` |
| **LeftInversed** | Инвертировать направление левого мотора | `False` |
| **RightInversed** | Инвертировать направление правого мотора | `False` |

### Шаг 2 — Блок синхронизации (запускается на каждой итерации цикла)

| Параметр | Описание |
|----------|----------|
| **E1** | Показание левого энкодера |
| **E2** | Показание правого энкодера |

---

## Режимы

| Режим | Описание |
|-------|----------|
| `Configuration` | Задать Kp, Power1, Power2, Ports, флаги инверсии (вызвать один раз) |
| `SYNC_Std` | Синхронизация с использованием настроенных Power1/Power2 |
| `SYNC_byPwr` | Синхронизация с Power1/Power2, передаваемыми на каждой итерации (переопределяет конфигурацию) |

---

## Типичная структура цикла

```
[Reset encoders]
[Configuration: Ports=B+C, Kp=0.12, Power1=60, Power2=60]

Loop:
  [Read Encoder B] → E1
  [Read Encoder C] → E2
  [SYNC_Std: E1, E2]
```

Или с динамической мощностью:

```
Loop:
  [Read Encoder B] → E1
  [Read Encoder C] → E2
  [SYNC_byPwr: Power1=speed, Power2=speed, E1, E2]
```

---

## Советы

- Установите `Power1 = Power2` для движения по прямой. Намеренно неравные значения обеспечат постоянную дугу.
- Используйте `LeftInversed` / `RightInversed`, если ваши моторы физически зеркально расположены (один направлен вперёд, другой назад).
- Увеличьте `Kp` для более точной синхронизации; если робот начинает колебаться, уменьшите его.
- Комбинируйте с **контроллером ускорения**, чтобы поддерживать синхронизацию моторов во время разгона и торможения: подайте `Output` в `SYNC_byPwr`.
